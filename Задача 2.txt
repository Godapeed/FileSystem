Написать сервис для пердачи данных о файловой системе со стороны сервера.
У сервиса есть следующие настройки:
1. Массив разрешенных путей. Можно указывать регулярные выражения.
Сервис будет возвращать данные только из этого списка и их дочерних.
2. Список невидимых путей. Можно указывать регулярные выражения.
Сервис не должен возвращать информацию об этих путях так, как будто бы их нет.
3. Список запрещенных путей. Можно указывать регулярные выражения.
Сервис не должен возвращать информацию об этих путях так, возвращать ошибку.

Сервис имеет метод REST API который принимает следующие данные:
{
	path: путь к папке/файлу информацию о котором нужно вернуть,
	mask: маска для получения дочерних, регулярное выражение,
	onlyFolders: true|false
	onlyFiles: true|false
}
Возвращает данные в формате:
{ data: null } - если такого пути нет
{ data: {error: 'доступ запрещен'} } - если не удается прочитать информацию или путь в настройке запрещенных.
{ data: {
		name: название папки/файла,
		isFolder: находится ли по указанному пути папка (false - файл),
		size: размер файла в байтах,
		created: дата создания,
		updated: дата-время последнего изменения,
		children: [
			...массив описаний дочерних объектов папки в том же формате...
		]
	}
}
Если при обработке произошла ошибка, то необходимо возвращать статус ответа больше 400 и выводить ошибку в { data: {error: 'сообщение об ошибке'} }
Собщения должны быть осознанными, т.е. обрабатывать возможные варианты ошибки выводить корректное понятное пользователю сообщение.
